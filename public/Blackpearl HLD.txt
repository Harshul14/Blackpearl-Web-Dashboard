**Explanation of Components and Flows:**

1.  **User:** The external actor who interacts with the BlackPearl system.
2.  **User Interface (UI):** The presentation layer where users initiate actions like connecting repositories, asking code questions, uploading meetings, inviting team members, and purchasing credits. It acts as the gateway to all BlackPearl functionalities.
3.  **Repository Connector:** Handles the initial step of linking a user's GitHub repository by taking a URL and token. It integrates with the external GitHub platform.
4.  **File Indexer:** Receives data from the Repository Connector and processes the files within the connected repository. **Crucially, this action consumes credits**, with each file using one credit. The indexed data is prepared for AI analysis.
5.  **Meeting Processor:** Manages the upload of meeting audio files in supported formats (MP3, WAV, M4A). It prepares this data for analysis by the AI Engine.
6.  **AI Engine:** This is the core processing unit powered by Google Generative AI. It performs two main analytical tasks:
    *   **Code Analysis:** It accesses the **Indexed Code Data** from the Knowledge Base to provide precise, context-aware answers to user queries about the codebase.
    *   **Meeting Analysis:** It receives meeting audio data from the Meeting Processor and extracts insights and action items.
7.  **Knowledge Base/Data Storage:** This component acts as the persistent storage for the system. It stores the **indexed code data** that the AI uses for context, as well as the **extracted meeting insights and action items**. It also has the capability to store saved AI answers.
8.  **Collaboration/Project Management:** This module enables team collaboration. It allows users to invite team members, provides features like commit log visualization (tracking commits from the linked GitHub repository), and supports general project management activities. It likely interacts with storage or other components to retrieve relevant project and commit data.
9.  **Billing/Credit Management System:** Manages the credit system. Users purchase credits securely via Stripe. This system tracks the user's credit balance and is triggered by the File Indexer to consume credits based on the number of files indexed. The cost is fixed per credit ($0.02).
